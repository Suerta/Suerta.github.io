安装yum

yum是什么？

yum（ Yellow dog Updater, Modified）是一个在 Fedora 和 RedHat 以及 SUSE 中的 Shell 前端软件包管理器。

基于 RPM 包管理，能够从指定的服务器自动下载 RPM 包并且安装，可以自动处理依赖性关系，并且一次安装所有依赖的软体包，无须繁琐地一次次下载、安装。

yum 提供了查找、安装、删除某一个、一组甚至全部软件包的命令，而且命令简洁而又好记。

linux文件系统

目录 用处
/bin： 是系统的一些指令。bin为binary的简写，主要放置一些系统的必备执行档例如:cat、cp、chmod df、dmesg、gzip、kill、ls、mkdir、more、mount、rm、su、tar等。
/sbin： 一般是指超级用户指令。主要放置一些系统管理的必备程式例如:cfdisk、dhcpcd、dump、e2fsck、fdisk、halt、ifconfig、ifup、 ifdown、init、insmod、lilo、lsmod、mke2fs、modprobe、quotacheck、reboot、rmmod、 runlevel、shutdown等。
/usr：
/usr/bin： 是后期安装的一些软件的运行脚本。主要放置一些应用软体工具的必备执行档例如c++、g++、gcc、chdrv、diff、dig、du、eject、elm、free、gnome、 gzip、htpasswd、kfm、ktop、last、less、locale、m4、make、man、mcopy、ncftp、 newaliases、nslookup passwd、quota、smb、wget等。
/usr/sbin： 放置一些用户安装的系统管理的必备程式例如:dhcpd、httpd、imap、in.*d、inetd、lpd、named、netconfig、nmbd、samba、sendmail、squid、swap、tcpd、tcpdump等。
/dev： 任何设备均以文件形式存在于该文件夹内(通过mount命令挂载成用户直接可用的文件系统)
/media： 挂载的可移动设备
/etc： 配置文件所在目录
/proc： 是一种内核和内核模块用来向进程(process) 发送信息的机制(所以叫做/proc)。这个伪文件系统让你可以和内核内部数据结构进行交互，获取 有关进程的有用信息，在运行中(on the fly) 改变设置(通过改变内核参数)。 与其他文件系统不同，/proc 存在于内存之中而不是硬盘上。
/tmp： 临时文件，关机时自动销毁
/var： 系统产生的不可自动销毁的缓存文件、日志记录。（系统和程序运行后产生的数据、不对外提供服务、只能用户手动清理）（包括mail、数据库文件、日志文件）

关机

在Linux的领域里关机是很少见的，除非迫不得已，一般情况不会关机。

关机命令：shutdown

总结：无论关机还是重启，都要先进行备份。即，执行sync命令。

在Linux中我们可以使用 ll 或 ls -l 命令来显示一个文件的属性以及文件的所属的用户和组。

实例中，boot文件的第一个属性用”d”标识。

  • 当为[ d ]则是一个目录
  • 当是[ - ]则是一个文件；
  • 若是[ l ]则表示为链接文档(link file)
  • 若是[ b ]则表示为装置文件里面的可供存储的接口设备(可随机存取装置)
  • 若为[ c ]则表示为装置文件里面的串行端口设备，例如键盘，鼠标(一次性读取设备)。

以上的字符中，除去第一个字符后，以三个为一组，均为[rwx]的三个参数的组合。其中[ r ]代表可读(read)、[ w ]代表可写(write)、[ x ]代表可执行(excute).要注意的是，这三个权限的位置不会改变，如果没有权限，就会出现[ - ]符号。

每个文件的属性由左边第一部分的10个字符来确定，第一个字符标识文件类型【0】,第一组三个字符标识属主权限【1 2 3】，第二组三个字符标识属组权限【4 5 6】，第三组三个字符标识其他用户权限【7 8 9】。

Linux的账号管理学习

实现用户账号的管理，要完成的工作主要有以下几个方面。

  • 用户账号的添加、删除和修改。
  • 用户口令的管理。
  • 用户组的管理。
命令 参数说明
-m 自动创建这个用户的主目录 /home/用户名
-c comment 指一段注解性描述
-g 指定用户所属的用户组
-G 指定用户所属的附加组
-s 指定用户登录shell
-u 指定用户的用户号
-d 指定用户主目录 如果该目录不存在，则同时使用-m选项创建主目录

拓展：Linux 链接的概念

F1是源文件，F2是F1的硬链接，F3是F1的软链接

硬链接：删除F2对F1没有影响，删除F1对F2也没什么影响。相当于同步数据，各自存在。硬链接是占内存的。相当于是允许一个文件具有多个路径，用户可以通过这种机制建立硬链接到一些重要文件上，防止误删。

软链接：即快捷方式，删除F3对F1没有什么影响，但是删除F1，F3就找不到对应的路径，如下图。但是当我重新创建一个名为F1的文件，F3又可以用了。
